# 需求分析文档

**生成工具**: Claude Sonnet 4.5
**生成日期**: 2026-02-06
**项目**: CadToolOnline - VSCode 在线 CAD 多体动力学建模工具

---

## 1. 项目背景

### 1.1 项目起源

CadToolOnline 是将桌面版 CADToolbox (基于 Qt/C++ + OpenCASCADE) 迁移到 Web 平台的项目，目标是作为 VSCode 插件运行，为 Modelica 开发者提供在线的多体动力学建模工具。

### 1.2 目标用户

- **主要用户**: Modelica 开发者、机械工程师
- **使用场景**:
  - 导入 CAD 模型 (STEP 格式)
  - 定义多体动力学系统 (刚体、运动副、驱动)
  - 导出为 Modelica 仿真代码

### 1.3 核心价值

- ✅ 无需安装独立软件，在 VSCode 中直接建模
- ✅ 跨平台支持 (Windows, macOS, Linux)
- ✅ 与 Modelica 开发工作流无缝集成

## 2. 功能需求

### 2.1 核心功能

#### F1: STEP 文件导入

**优先级**: P0 (必须)

**需求描述**:
- 支持导入 STEP 格式的 CAD 文件
- 在浏览器端解析（无需服务器）
- 显示导入进度
- 支持文件大小 ≤ 100MB

**验收标准**:
- [x] 能够成功导入标准 STEP 文件
- [x] 导入进度实时显示
- [ ] 支持错误文件的友好提示
- [ ] 大文件 (>50MB) 导入性能优化

#### F2: 3D 可视化

**优先级**: P0 (必须)

**需求描述**:
- 使用 Three.js 渲染 3D 模型
- 支持旋转、缩放、平移操作
- Fit View (自动调整视图适配模型)

**验收标准**:
- [x] 渲染质量达到可用水平
- [x] 相机控制流畅（60 FPS）
- [x] Fit View 功能正常
- [ ] 支持选择高亮

#### F3: 刚体组定义

**优先级**: P0 (必须)

**需求描述**:
- 从导入的几何体创建刚体组 (MbsGroup)
- 自动计算质量属性（质量、质心、惯性矩阵）
- 支持手动编辑质量属性

**验收标准**:
- [x] 能够创建刚体组
- [ ] 质量属性计算准确（与桌面版对比误差 <1%）
- [ ] 支持密度设置
- [ ] 支持手动覆盖质量属性

#### F4: 坐标系与标记点

**优先级**: P1 (重要)

**需求描述**:
- 在刚体上定义坐标系 (MbsFrame)
- 定义标记点 (MbsMarker)
- 支持相对位置和姿态编辑

**验收标准**:
- [ ] 能够创建坐标系
- [ ] 坐标系可视化（三色轴）
- [ ] 支持位置和姿态编辑
- [ ] 坐标系相对于刚体固定

#### F5: 运动副定义

**优先级**: P0 (必须)

**需求描述**:
- 支持 7 种运动副类型：
  - Revolute (转动副)
  - Prismatic (移动副)
  - Cylindrical (圆柱副)
  - Spherical (球副)
  - Universal (万向节)
  - Planar (平面副)
  - Fixed (固定副)
- 基于两个坐标系定义运动副
- 可视化运动副约束

**验收标准**:
- [ ] 支持所有 7 种运动副类型
- [ ] 运动副参数可编辑
- [ ] 约束合法性检查
- [ ] 可视化显示约束关系

#### F6: Modelica 导出

**优先级**: P0 (必须)

**需求描述**:
- 将 MBS 模型导出为 Modelica 代码
- 支持 Modelica Multibody 库格式
- 生成可直接仿真的 .mo 文件

**验收标准**:
- [ ] 导出的 Modelica 代码语法正确
- [ ] 能在 OpenModelica / Dymola 中加载
- [ ] 质量属性、运动副参数正确转换
- [ ] 坐标系转换正确

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标 |
|------|------|
| STEP 文件导入速度 | <10s (10MB 文件) |
| 3D 渲染帧率 | ≥60 FPS (中等复杂模型) |
| WASM 模块加载时间 | <3s |
| 插件启动时间 | <2s |

### 3.2 兼容性要求

| 平台 | 最低版本 |
|------|----------|
| VSCode | 1.85.0 |
| Node.js | 20.0.0 |
| Chrome | 最新版 |
| Firefox | 最新版 |
| Safari | 最新版 |

### 3.3 安全性要求

- ✅ 所有文件操作使用 VSCode API（安全沙箱）
- ✅ WASM 模块不处理敏感信息
- ✅ 文件大小限制，防止 DoS 攻击
- ✅ 输入验证，防止注入攻击

### 3.4 可维护性要求

- ✅ 代码覆盖率: 核心模块 >80%, UI 模块 >60%
- ✅ 模块化设计，职责清晰
- ✅ 完善的 TypeScript 类型定义
- ✅ 遵循编码规范（见 .claude/rules/coding-style.md）

### 3.5 可扩展性要求

- ✅ 支持插件式添加新的运动副类型
- ✅ 支持多种导出格式（当前仅 Modelica，未来可扩展）
- ✅ 支持多种 CAD 格式（当前仅 STEP，未来可扩展 IGES, STL）

## 4. 约束条件

### 4.1 技术约束

- **几何内核**: 必须使用 OpenCASCADE (WASM)
- **渲染引擎**: 必须使用 Three.js
- **运行环境**: 必须在 VSCode WebView 中运行
- **构建工具**: 必须使用 pnpm workspace

### 4.2 业务约束

- **开源协议**: 内部项目，暂不开源
- **参考项目**: 保持与桌面版 CADToolbox 功能一致
- **文件格式**: 优先支持 STEP（行业标准）

### 4.3 资源约束

- **WASM 模块大小**: <20MB (压缩后)
- **内存占用**: <500MB (中等复杂模型)
- **文件大小限制**: 100MB
