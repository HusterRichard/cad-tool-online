## 功能概述

为了让导入的 STEP 模型更加美观，我们实现了一个智能的随机颜色生成系统。该系统使用**黄金角分割（Golden Ratio）**算法，确保生成的颜色：

- ✅ 视觉上协调且美观
- ✅ 色彩分布均匀
- ✅ 尽可能多样化
- ✅ 避免相邻零件颜色过于相似

## 技术实现

### 核心算法：黄金角分割

```cpp
class SmartColorGenerator {
private:
    int colorIndex = 0;
    static constexpr double GOLDEN_RATIO_CONJUGATE = 0.618033988749895;
    static constexpr double DEFAULT_SATURATION = 0.75;  // 饱和度 75%
    static constexpr double DEFAULT_VALUE = 0.90;       // 明度 90%

public:
    std::string getNextColor() {
        // 使用黄金角分割生成均匀分布的色相
        double hue = std::fmod(colorIndex * GOLDEN_RATIO_CONJUGATE, 1.0);

        // 轻微变化饱和度和明度增加多样性
        double saturation = DEFAULT_SATURATION + (variation1);
        double value = DEFAULT_VALUE + (variation2);

        // HSV → RGB 转换
        int r, g, b;
        hsvToRgb(hue, saturation, value, r, g, b);

        colorIndex++;
        return hexColor;
    }
};
```

### 为什么使用黄金角分割？

黄金角分割（φ ≈ 0.618）是自然界中最优雅的数学比例之一，用于生成色相时可以确保：

1. **均匀分布**: 颜色在色环上均匀分布
2. **避免重复**: 不会出现相近的颜色聚集
3. **视觉协调**: 符合人眼的审美习惯

## 技术细节

### HSV 色彩空间

我们使用 HSV 而不是直接的 RGB，因为：

1. **直观控制**:
   - H (色相): 0-360° 控制颜色种类
   - S (饱和度): 0-100% 控制鲜艳程度
   - V (明度): 0-100% 控制亮度

2. **均匀分布**:
   - 在 HSV 空间中旋转色相，颜色分布更均匀
   - 在 RGB 空间中随机生成，容易产生相似颜色

3. **视觉协调**:
   - 固定饱和度和明度，只变化色相
   - 确保所有颜色的"强度"一致

### HSV → RGB 转换公式

```
给定 H ∈ [0,1], S ∈ [0,1], V ∈ [0,1]

1. C = V × S        (色度)
2. X = C × (1 - |H×6 mod 2 - 1|)
3. m = V - C

4. 根据 H 的区间选择 (R', G', B'):
   H ∈ [0, 1/6):   (C, X, 0)
   H ∈ [1/6, 2/6): (X, C, 0)
   H ∈ [2/6, 3/6): (0, C, X)
   H ∈ [3/6, 4/6): (0, X, C)
   H ∈ [4/6, 5/6): (X, 0, C)
   H ∈ [5/6, 1):   (C, 0, X)

5. (R, G, B) = ((R'+m)×255, (G'+m)×255, (B'+m)×255)
```

## 与原有颜色兼容性

```cpp
if (hasColor) {
    // 优先使用 STEP 文件中的原始颜色
    result << ",\"color\":\"" << stepFileColor << "\"";
} else {
    // 无颜色时使用智能随机颜色
    if (isSimpleShape || nodeType == "part") {
        result << ",\"color\":\"" << g_colorGenerator.getNextColor() << "\"";
    } else {
        // 装配体使用中性灰色
        result << ",\"color\":\"#C0C0C0\"";
    }
}
```

**策略**:
1. ✅ 有颜色 → 保留原始颜色
2. ✅ 无颜色 + 实体零件 → 智能随机颜色
3. ✅ 无颜色 + 装配体 → 中性灰色

## 对比分析

### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **固定灰色** | 简单、统一 | 单调、难以区分零件 |
| **纯随机 RGB** | 多样化 | 颜色可能混乱、不协调 |
| **黄金角分割** ✅ | 多样化、协调、美观 | 略微复杂（可忽略） |

### 视觉效果对比

