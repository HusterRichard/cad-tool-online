# CadToolOnline Plan

## 1. 项目目标

将桌面版 CADToolbox 的核心能力迁移到 VSCode + Web 技术栈，形成可用于多体动力学（MBS）建模的在线化 CAD 工具。

核心目标：
- 支持 STEP 文件导入、解析与 3D 可视化。
- 支持 MBS 基础建模（刚体组、坐标系、运动副、驱动）。
- 支持导出 Modelica 仿真代码。
- 在可维护的 Monorepo 架构下持续迭代。

## 2. 范围定义

### 2.1 In Scope
- VSCode 插件形态的编辑器能力。
- OpenCASCADE WASM 几何解析。
- Three.js 交互与可视化。
- MBS 领域模型与业务编排。
- 单元测试、基础集成测试、构建与打包。

### 2.2 Out of Scope（当前阶段）
- 云端协作与实时多人编辑。
- 全量 CAD 格式兼容（除 STEP 外）。
- 企业级权限系统与多租户。

## 3. 分阶段计划

## Phase 1：基础设施与工程化（已完成）
目标：建立可持续开发基础。
交付物：
- pnpm workspace Monorepo。
- TypeScript 工程配置。
- 基础脚本（run/test/setup）。
- ESLint + Prettier 规范。
验收标准：
- `pnpm install`、`pnpm build` 成功。
- 主要模块可独立构建。

## Phase 2：OCCT WASM 能力接入（已完成）
目标：建立几何内核能力。
交付物：
- Emscripten + OCCT 构建链路。
- STEP 文件读取与层级输出。
- 网格数据输出用于渲染。
验收标准：
- `pnpm build:wasm` 成功。
- 至少 3 个 STEP 示例可正常导入显示。

## Phase 3：核心建模能力（进行中）
目标：形成可用的 MBS 建模主链路。
交付物：
- MbsGroup/MbsFrame/MbsMarker/MbsJoint/MbsMotion 数据模型。
- 基础交互：选择、编辑、创建。
- 运动副定义（优先 3 种高频类型，再扩展至 7 种）。
验收标准：
- 能完成一个典型机械装配体的基础建模流程。
- 核心数据结构可序列化/反序列化。

## Phase 4：VSCode 体验完善（进行中）
目标：补齐“可用产品”体验。
交付物：
- 面板交互优化（模型树、属性面板、工具栏）。
- 命令面板入口补齐与快捷操作。
- 错误提示、日志与调试信息标准化。
验收标准：
- 新用户可在 15 分钟内完成导入并进行基础编辑。
- 常见错误有可理解提示与处理建议。

## Phase 5：测试、性能与发布（待启动）
目标：形成稳定发布能力。
交付物：
- 单元测试、集成测试补齐。
- 大文件导入与渲染性能优化。
- VSIX 打包与发布流程固化。
验收标准：
- Core/Geo 测试覆盖率 >= 80%。
- Three/UI 覆盖率 >= 60%。
- 标准样例文件导入与渲染性能达标（见 KPI）。

## 4. 里程碑与 KPI

## M1：基础可用版
定义：可导入 STEP、可浏览模型、可进行基础选择与属性查看。
KPI：
- STEP 导入成功率 >= 95%（标准样例集）。
- 首屏渲染时间 <= 5s（中等模型，参考开发机）。

## M2：建模工作版
定义：支持主要 MBS 元素创建与编辑。
KPI：
- 典型建模流程完成率 >= 90%（内部测试脚本）。
- 关键操作错误率 < 2%。

## M3：发布候选版
定义：具备稳定性、测试覆盖与可交付性。
KPI：
- 自动化测试通过率 100%。
- VSIX 打包和安装流程稳定可重复。

## 5. 工作分解（WBS）

- 几何内核：WASM 构建、STEP 解析、数据桥接。
- 渲染层：网格加载、材质/颜色、选择高亮、相机控制。
- 业务层：MBS 对象生命周期、约束关系、数据校验。
- 插件层：命令、面板、状态同步、错误处理。
- 质量保障：测试、性能基线、回归机制。
- 文档交付：README、Quick Start、API 与变更记录。

## 6. 风险与应对

- OCCT/WASM 编译复杂度高：
  - 应对：固定工具链版本，沉淀可重复脚本与故障排查文档。
- 大模型性能瓶颈：
  - 应对：分块加载、LOD、异步处理（Worker）。
- VSCode Webview 交互复杂：
  - 应对：建立明确事件协议与状态同步规范。
- 测试数据不足：
  - 应对：扩充 `tests/fixtures`，建立分层样例库。

## 7. 质量与验收策略

- 每个阶段必须包含：
  - 可运行演示路径。
  - 对应测试用例或验证清单。
  - 文档更新（至少包含变更说明与使用方式）。
- 合并门禁建议：
  - Lint + Type Check 必过。
  - 关键路径测试必过。
