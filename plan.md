# CadToolOnline Development Plan（基于 PRD v2.0）

## 1. 计划目标

基于 `docs/outputs/PRD_VSCode_CADTool_Autocomplete_2026a.md`，交付可替代 CADTool 核心流程的在线三维建模系统，覆盖：

- CAD 模型导入
- 多体设计
- 流体设计
- 设计工具
- 模型导出
- 迭代设计

明确不做：

- Modelica Language Server
- `.mo` 文件打开/编辑能力
- IDE 自动补全能力

### WP-MB-01 偏差分析（2026-03-06）

当前状态（证据）：
- 已有：分组创建/子分组/分组属性、Marker 创建与删除（`createGroup`/`createChildGroup`/`createFrame`/`deleteFrame`）。
- 未完成：参考标架能力未见独立实现，`editFrame` 仍为提示占位；设计点（designPoint）仅见语言层关键字，运行时未见实体与流程。

未启动/推进不足原因：
- 范围被导入链路、配置导入导出、语言特性与 QA 回归占用，MBS 子域深度功能被后置。
- 领域模型未先行拆解（参考标架、设计点缺少独立实体/状态/导入导出映射），导致只能做 UI 级动作。
- 验收口径未拆成可执行子任务（“分组/标架/参考标架/设计点”被合并为一条），完成判定过早。

纠偏动作（下个迭代）：
- [ ] 拆分 WP-MB-01 子任务：`group`、`frame`、`reference frame`、`designPoint` 四条独立验收项。
- [ ] 补齐 `reference frame` 与 `designPoint` 运行时实体、属性面板、导入导出映射。
- [ ] 将 `editFrame` 从占位提示改为可编辑流程（位置/方向/挂载对象）。
- [ ] 增补多体回归用例：覆盖“创建-编辑-导出-再导入”闭环后再勾选完成。

## 6. 测试与质量门禁

每个 Sprint 的合并门禁：

1. 功能分支自测清单通过
2. 单元测试/集成测试通过
3. 关键路径回归（导入 -> 设计 -> 导出）通过
4. 文档与变更说明同步更新

里程碑验收门禁：

1. 与 PRD FR 对应项逐条验收
2. 至少通过 3 个官方案例回放
3. 已知高风险问题有规避或修复方案

## 7. 资源与角色建议

- 产品：1（需求澄清、范围控制、验收）
- 前端/3D：2-3（UI、三维交互、性能）
- 领域开发：2（多体/流体/导出/迭代）
- 测试：1-2（功能回归、性能验证）
- 技术负责人：1（架构与质量门禁）

## 8. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 大模型渲染/拾取性能不足 | 交互卡顿 | 分层加载、LOD、异步任务拆分 |
| 几何拾取规则复杂 | 行为偏差 | 优先实现高频规则，案例驱动回归 |
| 商用导入依赖环境 | 导入失败 | 增加依赖预检查与引导提示 |
| 迭代设计冲突多 | 解析失败 | 显式冲突检测 + 可回退路径 |

## 9. 交付物清单

1. 功能代码（各 WP 对应模块）
2. 自动化测试与回归用例
3. 发布说明与用户操作指南
4. 验收报告（FR 覆盖 + 案例结果 + 已知问题）

## 10. 计划维护机制

- 更新频率：每个 Sprint 结束更新一次 `plan.md`
- 更新内容：
  - 完成项/延期项
  - 风险状态变化
