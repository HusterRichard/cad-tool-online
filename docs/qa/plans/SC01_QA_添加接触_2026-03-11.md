## 8. 关键测试场景

### 8.1 入口与状态

1. 点击 Ribbon `点点接触`，右侧立即显示 `选项-接触`。
2. 点击 Ribbon `点面接触`，右侧立即显示 `选项-接触`，且默认类型与入口一致。
3. 未加载模型时点击接触入口，系统提示 `Please load a model first`，且不进入接触创建态。
4. 在接触设计状态下再次点击同类接触入口，或按 `Esc`，系统退出设计状态。

### 8.2 模式切换

1. 首次进入默认是 `闪电模式`。
2. 切换到 `标准模式` 后，名称、类型、图标大小输入值不丢失。
3. 从 `标准模式` 切回 `闪电模式` 后，草稿状态不异常。

### 8.3 创建流程

1. 闪电模式下，依次拾取两个对象后立即创建接触。
2. 标准模式下，依次拾取两个对象后不会立即创建，点击 `添加` 后才创建。
3. 若两次拾取来自同一零件，系统阻止创建并给出提示。
4. 创建成功后，三维视图出现绿色接触图标。
5. 创建成功后，模型树 `力` 分类新增接触节点。
6. 连续创建两个以上接触时，系统保持在接触设计状态，不强制退出。

### 8.4 属性与选择

1. 在模型树中单击接触节点，右侧显示 `属性-接触`。
2. 属性面板中至少可看到：
   - 名称
   - 类型
   - 零件 1
   - 零件 2
   - 图标大小
   - 可见性
   - 创建时间
3. 在三维视图中选中接触对象时，关联零件呈现红/蓝差异着色。

### 8.5 删除

1. 删除接触后，模型树中的接触节点消失。
2. 删除接触后，三维视图中的接触图标同步消失。
3. 删除一个接触不会影响其他接触对象。

### 8.6 导入导出

1. 导出配置后，生成的 JSON 中包含 `contact` 数组。
2. 清空场景后重新导入配置，接触对象数量与名称恢复一致。
3. 导入包含未知 `partA / partB` 的接触配置时：
   - 系统不崩溃
   - 给出 warning
   - 对象仍可保留原始引用值
