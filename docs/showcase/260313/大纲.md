# CadToolOnline AI 编程七步法分享大纲

## 背景

我现在准备做一个技术分享汇报，计划基于配图 `docs/showcase/260313/7-phase.png` 的七步法，分享本项目中的 AI 编程经验。

## 七步法

### 1. Idea 创意想法

- 参考 `TASK.md`。
- `CADTool` 是一个桌面版软件，其产品简介可在该文档中找到。
- 当前目标是将其 Web 化。
- 主要难点在于该工具基于几何引擎 `OCCT` 开发，原始技术体系是 `C++/Qt`，因此 Web 化存在明显技术门槛。

### 2. Research 研究

- `chili3d` 是一个可参考的开源项目。
- 它通过 `WebAssembly` 技术将 `OCCT` 几何引擎编译为 `wasm` 文件，从而实现在浏览器中直接调用几何引擎进行计算。
- 基于这部分研究输入，项目最终形成了产品技术方案和软件架构设计。
- 相关文档：
  - `CLAUDE.md`
  - `.claude/agents/project-context.md`

### 3. Prototype 原型设计

- 基于前两步的工作，我们已经完成了软件架构搭建，并实现了 CAD 模型导入和三维可视。
- 原型设计部分采用的方案，是直接基于 CADTool 的产品帮助文档生成全量 `pencil` 格式的 UI/UX。
- 原型文件路径：`cadtoolonline.pen`
- 当前原型整体已经达到较高保真度。

### 4. PRD 产品详细设计

- 基于 CADTool 产品帮助文档，输出了 CadToolOnline 的全局详细设计文档：
  - `docs/prd/global/GLOBAL_PRD_CADToolOnline_三维建模界面重建_2026-03-06.md`
- 由于 CADTool 是一个复杂的三维 CAD 交互设计软件，因此进一步拆分出了 `SC01-SC60` 共 60 个原型场景。
- 在全局 PRD 基础上，后续工作采用“按场景迭代开发与验证”的方式推进。
- 我为此创建了基于场景开发的 skills。
- 以 `SC01` 为例，输入 `SC01` 后，会从全局 PRD 一路推进到“多体设计-添加接触”这个场景的：
  - 场景 PRD
  - 场景计划
  - QA 计划
  - 实现代码
  - 回归测试
  - 验证结果

### 5. Kanban 看板计划

- 该步骤与第 4 步配套推进。
- 在完成 PRD 设计的同时，AI 模型会生成：
  - 整体里程碑计划
  - 具体场景的详细开发计划，例如 `SC01`
- 项目通过看板跟踪开发进度。
- 看板计划可以在开发过程中持续迭代修正。
- 相关材料位于 `docs/plan` 文件夹下的多个 plan 文件。

### 6. Execute 开发执行

- 该步骤与第 4 步、第 5 步并行开展。
- 具体方式是基于 PRD 设计，按照 Kanban 计划严格执行，并在开发过程中不断迭代，直到满足详细设计要求。
- 本质上，这是一个 `PRD 文档`、`Plan 文档` 与 `代码实现` 同步迭代开发的过程。
- 基本规律是：当 PRD 文档已经能够准确描述产品功能时，代码开发通常也已经接近完成。
- 这一流程已经沉淀到 `.codex/skills/cad-scene-delivery` 这个 skill 中。

### 7. QA 测试

- QA 与第 4 步到第 6 步同步推进。
- 在迭代开发过程中，会同步生成：
  - 单元测试
  - 集成测试
  - E2E 测试
- 相关测试说明见 `tests/README.md`
- 每次修改代码后，都会执行对应影响范围内的测试代码，以保证改动可验证、可回归。
