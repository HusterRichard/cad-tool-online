## 3. 背景与目标

加载 CAD 模型后，用户需要先把“不产生相对运动”的零件整理成分组，后续的标架、参考标架、设计点、连接、驱动、检查和导出，都会以分组或分组下零件为基础继续设计。`SC13` 的目标不是单一“创建分组”按钮，而是完整的分组工作流。

根据官方帮助页，分组场景至少包括两类主动作:

- 组合零件
- 移动零件

并进一步覆盖:

- 分解分组
- 清理空分组
- 添加默认分组
- 删除零件/分组

因此，`SC13` 的 PRD 需要同时定义:

- 帮助文档要求的目标态交互与规则
- 当前仓库中已经落地的能力边界

## 4. 用户与典型场景

### 4.1 目标用户

- 需要把 CAD 装配体整理为多体系统刚体组的建模工程师
- 需要在添加连接和驱动前完成对象归属梳理的仿真工程师
- 需要迭代调整导入后分组结构的 CADToolOnline 用户

### 4.2 典型场景

- 从模型树或三维视图多选零件并组合为新的刚体组
- 将某个零件从当前分组拖拽到另一个分组，或拖回根节点变为未分组
- 对自动生成的默认分组进行分解，再重新组织分组结构
- 选中某个分组后，通过`分解`删除该分组并将组内零件回收到根节点`物体`
- 为未分组零件批量创建默认分组，作为后续设计起点
- 删除无依赖的零件或空分组

## 6. 目标态产品要求

以下内容来自帮助页与案例页，代表 `SC13` 的目标态要求。

### 6.1 组合零件

- 用户可通过两类入口进入“添加分组”状态:
  - 单击 Ribbon 的`组合`
  - 在模型浏览器或三维视图中右击后选择`组合`
- 进入状态后，右侧弹出`选项-组合`侧边栏
- 用户输入分组名称后，单击`添加`即可将当前选中零件组合为一个新分组
- 单击`取消`、按 `Esc` 或再次点击入口，可退出该状态

### 6.2 移动零件

- 用户可通过两类方式移动:
  - 右击选择`移动至`并指定目标分组
  - 在模型浏览器中拖拽节点
- 支持将零件移入目标分组
- 支持将零件移到根节点，恢复为未分组状态
- 拖拽分组前应先退出分组状态
- 不允许把对象移动到非法目标

### 6.3 分解分组

- 用户可通过 Ribbon 的`分解`操作拆开选中的分组
- 被分解分组下的零件与子分组会平铺回根节点`物体`
- 默认分组是常见的分解目标

### 6.4 清理空分组

- 在零件移动、重新组合后，原有空分组可能残留
- 用户可通过 Ribbon 的`清理`一键删除空分组
- 该动作只应删除空分组，不应破坏非空组
- 若存在被标架、参考标架、设计点等对象引用的空分组，系统应跳过并给出提示

### 6.5 默认分组

- 当顶层或子物体下存在较多未分组零件时，用户可单击`默认分组`
- 系统应仅为当前未分组零件创建默认分组，已在其它分组中的零件不得被重复迁入默认分组
- 默认分组创建后应默认选中新建分组，便于用户继续查看或调整属性

### 6.6 删除零件 / 分组

- 删除入口位于:
  - 模型浏览器右键菜单
  - 三维视图右键菜单
  - 键盘 `Delete`
- 删除后模型树与三维视图都应同步更新
- 官方帮助页明确指出删除后应支持撤销与重做

### 6.7 命名规则

- 分组名称必须满足 Modelica 命名规范
- 分组名称不能与其它物体或设计元素重名
- 若名称重复，系统需自动规整，规则为在名称后追加 `_` 与递增序号

## 8. 核心交互流程

### 8.1 组合零件主流程

1. 用户在模型树或三维视图中选择一个或多个零件。
2. 用户点击`组合`或右击选择`组合`。
3. 右侧打开`选项-组合`。
4. 用户输入组名并确认。
5. 系统创建新分组，并把选中零件迁入该组。
6. 模型树刷新并默认选中新组。

### 8.2 移动零件 / 分组主流程

1. 用户选择一个或多个零件，或选择一个/多个分组。
2. 用户通过`移动到...`或树拖拽发起移动。
3. 系统校验目标是否合法。
4. 系统更新源组、目标组和未分组集合。
5. 模型树与属性面板同步刷新。

### 8.3 分解分组主流程

1. 用户选中一个分组。
2. 用户点击`分解`。
3. 系统校验该分组是否被设计元素引用。
4. 若合法，系统将子分组和零件回挂到根节点`物体`。
5. 原分组被删除。

### 8.4 清理空分组主流程

1. 用户点击 Ribbon `清理`。
2. 系统递归识别无成员、无子组的空分组。
3. 系统跳过被标架、参考标架、设计点等对象引用的空分组。
4. 系统删除其余空分组并反馈数量。

### 8.5 默认分组主流程

1. 用户点击`默认分组`。
2. 系统检查当前是否存在未分组零件。
3. 若存在，则只将这些未分组零件迁入新建默认分组。
4. 若不存在，则提示没有可处理对象。

### 8.6 删除主流程

1. 用户选择零件或分组。
2. 用户通过右键菜单或 `Delete` 触发删除。
3. 系统检查依赖、叶子节点限制和分组状态。
4. 合法对象被删除，非法对象给出阻塞原因。
5. 模型树、三维视图、属性面板同步刷新。

## 9. 功能需求

### 9.1 FR-01 分组入口

- 系统必须在 Ribbon 提供`组合`、`分解`、`清理`、`默认分组`入口
- 系统必须在模型浏览器右键菜单提供分组相关操作

### 9.2 FR-02 多选与目标选择

- 系统必须支持多个零件组成新分组
- 系统必须支持多个零件或多个分组移动
- 系统必须支持通过目标分组下拉框或树拖拽指定目标位置

### 9.3 FR-03 分组命名

- 系统必须校验分组名称是否合法
- 系统必须对非法名称自动规范化
- 系统必须对重名分组自动追加序号，避免冲突

### 9.4 FR-04 分组树

- 系统必须在模型浏览器中展示分组层级
- 系统必须同时保留未分组零件显示
- 系统必须支持分组节点展开/收起
- 系统必须在 STEP 导入完成后将模型树所有节点初始化为折叠状态

### 9.5 FR-05 移动规则

- 系统必须支持 `part -> group`
- 系统必须支持 `part -> root`
- 系统必须支持 `group -> group`
- 系统必须禁止 `group -> 自身`
- 系统必须禁止 `group -> 后代组`

### 9.6 FR-06 分解规则

- 系统必须支持将分组内零件和子分组平铺回根节点`物体`
- 系统必须在被设计元素引用时阻止分解并给出提示

### 9.7 FR-07 清理空分组

- 系统必须只删除空分组
- 系统必须跳过被引用的空分组
- 系统不得误删非空组

### 9.8 FR-08 默认分组

- 系统必须仅处理未分组零件
- 系统必须将默认分组标记为 `default` 类型
- 系统不得把已属于其它分组的零件再次迁入默认分组

### 9.9 FR-09 删除保护

- 系统必须允许删除无依赖叶子零件
- 系统必须允许删除无成员、无子组、无引用的分组
- 系统必须阻止删除被引用对象，并显示阻塞原因

### 9.10 FR-10 层级导入导出

- 系统必须支持把分组层级结构导出为可持续编辑的数据格式
- 系统必须支持再次导入该结构并恢复父子层级、分组类型和顺序

### 9.11 FR-11 关联对象保护

- 系统至少应保护以下已建对象对分组的引用:
  - 标架
  - 参考标架
  - 设计点

### 9.12 FR-12 模型树与三维视图联动反馈

- 当用户在模型树中选中某个分组时，系统必须在三维视图中高亮该分组下全部零件，且高亮效果应与模型树中直接选中这些零件时保持一致
- 当用户在模型树中悬停某个零件或分组时，系统必须在三维视图中显示对应对象的立方体包围盒
- 对分组的包围盒计算必须递归覆盖其全部子分组和成员零件；对零件的包围盒计算则仅覆盖该零件自身

## 10. 异常与边界条件

| 场景 | 期望处理 |
|---|---|
| 未选择任何零件/分组就发起组合或移动 | 弹出“请先选择对象”提示 |
| 分组名为空或包含非法字符 | 自动规范化为合法名称 |
| 分组名重复 | 自动追加 `_1`、`_2` 等后缀 |
| 把分组移动到自身或后代 | 阻止操作并提示非法目标 |
| 分解被引用分组 | 阻止操作并提示存在设计元素引用 |
| 清理时存在被引用空分组 | 跳过并提示 |
| 点击`默认分组`时部分零件已在其它分组内 | 仅处理未分组零件，已分组零件保持不变 |
| 删除非叶子零件 | 阻止删除 |
| 删除仍有成员或子组的分组 | 阻止删除 |
| 删除被设计元素引用的对象 | 阻止删除并提示引用来源 |

## 11. 验收标准

### 11.1 功能验收

- 用户可通过 Ribbon 或右键菜单创建分组
- 用户可将零件移动到组内或移回根节点
- 用户可分解分组，并看到原分组成员回到根节点`物体`
- 用户可清理空分组
- 用户可创建默认分组
- 用户可删除满足条件的零件或空分组

- 模型树能够显示真实分组层级
- 导入 STEP 后模型树默认折叠

## 12. 设计基线与交付快照

### 12.1 架构基线

- 导入 CAD 装配树与可编辑分组树保持为两套结构:
  - CAD 树用于反映源几何层级
  - Group 树用于承载多体设计归属关系
- 分组状态以层级模型为核心，最少包含:
  - `groupsById`
  - `rootGroupIds`
  - `ungroupedPartIds`
  - `selectedNodeIds`
  - `activeMode`
- 分组节点最少包含:
  - `id`
  - `name`
  - `parentGroupId`
  - `childGroupIds`
  - `memberPartIds`
  - `kind`
  - `order`
  - `createdAt`
- 分组配置导出除 `name` / `parts` 外，还应保留:
  - `parentRef`
  - `kind`
  - `order`

### 12.2 当前交付快照

- 当前 Webview 已启用层级化 group state，而不再依赖“最后一个分组”式归属判断
- 模型树已支持 group 节点、Ctrl 多选、右键菜单、拖拽和拖放到根节点
- 核心命令已覆盖:
  - 创建分组
  - 创建子组
  - 重命名
  - 默认分组
  - 清理空分组
  - 移动对象
  - 分解分组
  - 删除安全对象
- 分组属性已覆盖基础统计与聚合物理量展示
- 标架 / 参考标架 / 设计点 已能在创建时解析 owner group 并保存 `groupId`
- 分组层级导入导出已支持继续编辑

### 12.3 已知交付边界

- 非空分组删除后自动平铺回父级尚未实现，当前只允许删除空且未被引用的分组
- Joint / Motion 的归属仍偏向 part，不是完整的 group-oriented 模型
- 当前自动化回归主要集中在 `packages/core`，缺少专门的 Webview 集成测试套件
- 官方帮助页要求的撤销/重做尚未看到明确产品化证据

- 当前分组引用保护已覆盖标架、参考标架、设计点，但未见对更完整对象集合的统一依赖管理框架
- 当前默认分组实现更偏向“把所有未分组零件并入一个新默认组”，与帮助页“按顶层或子物体下未分组零件补组”的更细粒度策略仍有演进空间

### 14.2 建议

- 将 `SC13` 视为当前多体设计里完成度最高的场景，继续作为后续对象归属能力的基线
- 在后续版本中补齐统一命令栈，以满足删除、移动、分解的撤销/重做
- 把分组引用保护扩展到连接、驱动、接触和导出依赖
- 为三维视图分组操作补齐更明确的产品证据和回归测试
- 针对模型树悬停触发的包围盒反馈，建议补充包围盒缓存、增量失效与悬停去抖策略，避免每次 hover 都递归遍历整组网格，降低当前计算与显示响应偏慢的问题

### 15.1 手工回归清单

前置条件:

- 打开 CAD 编辑器并导入一个包含多个零件的 STEP 模型。
- 保证模型树左侧可见 `物体` 分类和至少 4 个可分组零件。

执行步骤:

1. 导入 STEP 后立即检查模型树。
   预期: `物体` 分类下节点默认全部折叠，未自动展开。
2. 在模型树中展开 `物体`，Ctrl 多选 2 个未分组零件，点击 Ribbon `组合`，输入组名并确认。
   预期: 新分组创建成功，被选中的 2 个零件进入该分组，新分组默认处于选中状态。
3. 保持该分组选中，检查右侧面板。
   预期: 面板标题为 `属性-零件`，且属性中的 `类型=分组`。
4. 保持步骤 2 创建的分组选中，观察三维视图。
   预期: 三维视图高亮该分组下全部零件，且高亮效果与模型树中直接选中这些零件一致。
5. 将鼠标依次悬停到步骤 2 创建的分组节点和其中一个零件节点。
   预期: 三维视图显示对应对象的立方体包围盒；分组包围盒覆盖该分组下全部零件，零件包围盒仅覆盖该零件自身。
6. 再选中 1 个未分组零件，点击 Ribbon `默认分组`。
   预期: 只把当前未分组零件放入新默认分组，步骤 2 已分组的零件位置不变。
7. 将一个未分组零件拖入步骤 2 创建的分组。
   预期: 拖拽成功，该零件从根节点消失并出现在目标分组下。
8. 将步骤 6 创建的默认分组拖到步骤 2 创建的分组下，形成父子分组。
   预期: 默认分组成为子分组，层级关系正确显示。
9. 选中步骤 2 创建的父分组，点击 Ribbon `分解`。
   预期: 父分组被删除；其直属零件和子分组都提升到根节点 `物体` 下。
10. 制造一个空分组后点击 Ribbon `清理`。
   预期: 只删除空分组；非空分组和其零件不受影响。
11. 在某个分组下创建 Marker / RefFrame / DesignPoint，再尝试分解该分组。
   预期: 系统阻止分解，并提示该分组已被引用。
12. 导出配置，清空场景后重新导入。
   预期: 分组层级、默认分组、未分组零件状态和属性面板行为均能正确恢复。

## 17. 2026-03-12 补充交互规则

本节为 `2026-03-12` 的增量补充说明。如与前文已有描述冲突，以本节规则为准。

### 17.1 同名零件与默认分组命名

- 当多个零件原始名称相同，且系统需要基于零件名自动生成分组名时，必须执行重名规整。
- 重名规整规则为在名称后追加递增后缀：`_1`、`_2`、`_3`……。
- 例如存在多个名为 `Bolt` 的未分组零件时，其自动生成的专属分组名称应依次为 `Bolt_1`、`Bolt_2`、`Bolt_3`，不得产生同名分组。

### 17.2 无所属分组零件的默认归属规则

- 导入CAD模型后，对于当前没有所属分组的零件，系统为每个零件分别创建一个专属父分组。
- 该专属父分组默认使用零件名称作为分组名称；若发生重名，则按 `17.1` 的 `_1`、`_2` 规则规整。
- 创建完成后，该零件成为对应专属父分组的唯一直接成员。

### 17.3 模型树展开状态保持

- 组合、分解、移动、清理空分组、默认分组、删除等导致模型树结构变化的操作完成后，系统应尽量保持用户当前的展开/折叠状态。
- 不得因为一次结构刷新而把模型树自动全部折叠。
- 对未受本次操作影响的节点，应保持原有展开状态；对新建节点，应在不破坏用户当前浏览上下文的前提下保证其可见。

### 17.4 STEP 初始导入后的默认展开规则

- 初始导入 STEP 后，模型树默认展开 `物体` 根节点。
- 在 `物体` 根节点下，如存在分组节点，则仅默认展开顶层分组节点。
- 顶层分组下的子分组节点、零件节点默认保持折叠，由用户按需逐级展开。
- 因此，前文中涉及“STEP 导入后全部默认折叠”的描述，均以本条补充规则为准。

### 17.5 Ribbon-分组-默认分组 的批量补组规则

- 用户执行 Ribbon `分组 -> 默认分组` 后，系统应遍历当前所有未分组零件。
- 系统必须为每一个未分组零件分别创建一个专属父分组，而不是创建单一“默认分组”来容纳多个零件。
- 专属父分组名称默认取零件名称；若重名，则按 `17.1` 追加 `_1`、`_2` 等后缀。
- 已经属于任意分组的零件不得被重复迁入新建默认分组。

### 17.6 模型树选中与悬停时的三维视图反馈

- 当用户在模型树中选中某个分组时，三维视图应高亮该分组下全部零件，且高亮效果应与模型树中直接选中这些零件时保持一致。
- 当用户在模型树中悬停某个零件或分组时，三维视图应显示对应对象的立方体包围盒。
- 对分组的包围盒计算必须递归覆盖其全部子分组和成员零件；对零件的包围盒计算则仅覆盖该零件自身。
- 包围盒反馈用于帮助用户快速确认对象空间范围，应尽量做到低延迟响应；如当前实现存在明显卡顿或显示延迟，后续版本应优先优化计算效率与渲染反馈链路。
