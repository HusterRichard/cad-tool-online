## 3. 背景与目标

在帮助页工作流中，接触属于多体设计阶段的核心对象，前置条件是零件已经完成分组。用户需要在三维视图中依次拾取两个零件的几何特征，快速建立接触关系，并在模型树、属性面板和配置导入导出中持续管理该对象。

`SC01` 的目标不是把 Ribbon 上两个“接触”按钮点亮，而是形成最小可用的接触设计闭环，包括：

- Ribbon 入口
- `选项-接触` 面板
- 闪电模式与标准模式
- 三维拾取两个零件创建接触
- 模型树中的 force 分类承载接触对象
- 属性查看与删除
- 配置导入导出

同时，本文明确区分两层要求：

- 帮助页目标态：以官方帮助页为准，描述最终应达到的完整交互
- 本次 MVP：以当前仓库可落地实现为准，优先交付接触对象闭环，不强行承诺复杂几何识别

## 4. 用户与典型场景

### 4.1 目标用户

- 已完成零件分组、需要继续进行多体设计的建模工程师
- 需要基于 CAD 几何面快速定义接触关系的仿真工程师
- 需要在模型树、三维视图、属性面板之间管理接触对象的 CADToolOnline 用户

### 4.2 典型场景

- 在两个已分组零件之间创建一个接触对象
- 连续创建多个接触对象并保持在设计状态中
- 在模型树中选择接触对象并查看其关联零件、类型和图标大小
- 删除误建的接触对象并重新创建
- 将接触对象随 `cadtool.config.json` 一起导出并重新导入

## 6. 帮助页目标态要求

以下内容直接来自 `BuildContacts.html`，代表帮助页定义的目标态。

### 6.1 进入方式

- 在零件分组完毕后，用户单击菜单栏 `接触`
- 右侧立即弹出 `选项-接触` 侧边栏
- 系统进入添加接触状态

### 6.2 闪电模式

- 默认模式为 `闪电模式`
- 用户先设置名称、接触类型、图标大小
- 用户在三维视图中依次选择：
  - `特征 1`：第一个零件的一个面
  - `特征 2`：第二个零件的一个面
- 选择完 `特征 2` 后立即创建接触
- 三维视图中绘制绿色接触图标，连接两个零件几何特征
- 用户可重复该流程连续创建多个接触
- 再次点击 `接触` 或按 `Esc` 退出

### 6.3 标准模式

- 在 `选项-接触` 底部关闭闪电模式后进入 `标准模式`
- 用户仍需依次选择 `特征 1` 与 `特征 2`
- 选择完成后不会立即创建
- 用户必须单击 `添加` 才完成创建
- 创建后继续保持接触设计状态，支持连续创建
- 再次点击 `接触` 或按 `Esc` 退出

### 6.4 几何规则

- 图标大小支持三种调节方式：
  - 输入值
  - 滑条
  - `Alt + 鼠标滚轮`
- 特征拾取时要进行蓝色高亮
- 接触类型限制如下：
  - `球球接触`：仅允许球面
  - `球与圆柱侧面接触`：仅允许一个球面和一个圆柱侧面
  - `球与矩形面接触`：仅允许一个球面和一个矩形面
  - `球与圆面接触`：仅允许一个球面和一个圆面
- 选中接触后：
  - 关联零件一红色高亮描边
  - 关联零件二蓝色高亮描边

## 7. 本次 MVP 产品要求

本次交付以当前仓库架构下的最小闭环为目标，优先实现“对象可创建、可查看、可删除、可导入导出”，对复杂几何限制采取后续迭代。

### 7.1 MVP 主流程

1. 用户点击 Ribbon 中的接触入口。
2. 右侧显示 `选项-接触`，系统进入接触设计状态。
3. 用户可设置：
   - 名称
   - 接触类型
   - 图标大小
   - 闪电模式开关
4. 用户在三维视图中依次拾取两个零件。
5. 系统记录零件 1、零件 2 及两次拾取位置。
6. 在闪电模式下，第二次拾取完成后立即创建接触。
7. 在标准模式下，第二次拾取完成后等待用户点击 `添加`。
8. 创建成功后：
   - 三维视图显示一个绿色接触图标/接触标识
   - 模型树 `力` 分类下新增接触节点
   - 右侧可切到 `属性-接触`
9. 用户可继续创建下一个接触，直到再次点击入口或按 `Esc` 退出。

### 7.2 MVP 交互规则

- 接触名称默认自动生成，如 `Contact1`、`Contact2`
- 接触类型至少覆盖当前 Ribbon 暴露的两类入口:
  - `point-point`
  - `point-surface`
- 图标大小支持数字输入；滑条与 `Alt + 滚轮` 可作为增强项，但推荐纳入本次实现
- 允许通过三维拾取完成零件 1 / 零件 2 选择
- 模型树选中接触后，右侧展示接触属性
- 删除接触后，模型树与三维显示同步刷新
- 导出配置时写入 `contact`
- 导入配置时恢复 `contact`

### 7.3 MVP 数据模型建议

接触对象至少应包含以下字段：

- `id`
- `name`
- `contactType`
- `part1`
- `part2`
- `pickPoint1`
- `pickPoint2`
- `iconSize`
- `visible`
- `createdAt`

## 8. 功能需求

### 8.1 FR-01 接触入口

- 系统必须在 Ribbon 中提供接触入口
- 接触入口不得再复用流体切片 / 流体端口命令
- 点击入口后必须切换到接触设计状态，并显示 `选项-接触`

### 8.2 FR-02 模式切换

- 系统必须支持 `闪电模式` 与 `标准模式`
- 默认进入 `闪电模式`
- 两种模式切换后，不得丢失已填写的名称、类型和大小

### 8.3 FR-03 三维拾取

- 系统必须支持在三维视图中依次拾取两个零件
- 系统必须记录两次拾取所对应的零件与位置
- 第二次拾取不得与第一次拾取的对象状态冲突

### 8.4 FR-04 接触创建

- 闪电模式下，第二次拾取后立即创建接触
- 标准模式下，只有点击 `添加` 才创建接触
- 创建成功后必须在模型树中生成接触节点
- 创建成功后必须在三维视图中生成接触可视对象

### 8.5 FR-05 属性展示

- 系统必须提供 `属性-接触` 面板
- 属性面板至少展示：
  - 名称
  - 类型
  - 零件 1
  - 零件 2
  - 图标大小
  - 创建时间

### 8.6 FR-06 删除

- 系统必须允许删除已创建接触
- 删除后模型树与三维视图必须同步移除

### 8.7 FR-07 导入导出

- 系统必须在 `cadtool.config.json` 导出中写入 `contact` 数组
- 系统必须支持导入 `contact` 数组并恢复对象
- 当 `part1/part2` 无法解析时，系统应给出 warning，并保留原始引用值

## 9. 异常与边界条件

| 场景 | 期望处理 |
|---|---|
| 未加载模型就点击接触 | 阻止进入创建流程并提示先加载模型 |
| 未完成分组即开始创建接触 | MVP 可允许直接基于零件创建，但需在文档中标明与帮助页目标态有偏差 |
| 只拾取了一个零件就退出 | 丢弃本次草稿，不创建对象 |
| 两次都拾取到同一零件 | 允许或阻止需产品明确；MVP 建议允许但给出提示，后续可升级为规则校验 |
| 标准模式未点击 `添加` 就切换模式 | 保留草稿 |
| 导入的接触引用未知零件 | 恢复对象并给出 warning |
| 删除接触后重新导出 | 导出结果中不再包含该对象 |

- 默认进入 `闪电模式`
- 可切换到 `标准模式`
- 可配置名称、类型、图标大小
- 可通过三维拾取两个零件创建接触
- 闪电模式下第二次拾取后立即创建
- 标准模式下必须点击 `添加` 后创建
- 创建成功后模型树 `力` 分类出现接触节点
- 选中接触后右侧显示 `属性-接触`

  - 接触入口

  - 闪电模式 / 标准模式
  - 三维拾取两个零件
  - 模型树 force 分类

## 11. 风险与后续项

- 当前仓库尚未形成严格的“面级特征类型识别”能力，球面 / 圆柱侧面 / 圆面 / 矩形面的限制短期内难以完整对齐帮助页。
- 三维视图对“零件一红描边、零件二蓝描边”的双对象差异化高亮尚未具备直接能力，需要扩展 viewer 选择管理器。
- 若接触对象后续需要与求解、导出 Modelica 语义强绑定，本次数据模型还需补充接触参数与约束语义。
