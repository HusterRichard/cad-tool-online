## 3. 背景与目标

连接是多体设计的核心建模能力。帮助文档明确要求用户在完成零件分组后，通过三维拾取快速建立固定副、转动副、平移副、圆柱副、球副、万向节、螺旋副和平面副，并在闪电模式与标准模式之间切换。

`SC02` 的目标不是“把 connector 写进 JSON”这么窄，而是建立一个可持续使用的连接工作流闭环:

- 进入连接命令后右侧出现 `选项-连接`
- 默认以闪电模式工作，也可切换到标准模式
- 用户可依次拾取 `零件 1` 和 `零件 2`
- 支持 `零件 2 = Ground`
- 固定副在已选 `零件 1` 后，支持通过三维空白处或模型树 `Ground` 直接完成与大地固定
- 支持智能推理开关
- 连接对象的位置和姿态默认由 `零件 1` 的拾取结果决定
- 标准模式支持位置、方向、`Z 轴反向` 二次修正
- 连接创建态显示白色 shape，创建成功后显示浅蓝色 shape
- 创建完成后，模型树与三维视图都能看到连接对象
- 模型树中的连接节点图标与对应 Ribbon 连接按钮保持一致
- 连接默认名称按连接类型分别编号，并优先使用该类型下一个可用编号，例如 `fixed_1`、`fixed_2`、`revolute_1`

## 4. 用户与典型场景

### 4.1 目标用户

- 需要在导入装配体后快速建立多体运动关系的仿真工程师
- 需要在遮挡较多的三维视图中连续创建连接的 CADToolOnline 用户

### 4.2 典型场景

- 将某一零件与 `Ground` 建立固定副
- 基于圆柱面快速建立转动副或平移副
- 在标准模式下调整连接位置和方向后再确认创建
- 通过万向节建立双方向约束关系

- 模型树中的连接节点与类型图标

- 8 类连接在三维中的独立线框形状

## 6. 界面要求

### 6.1 Ribbon

- 在“连接”分组提供 8 个连接按钮
- 再次点击可重新进入对应连接类型的创建状态

### 6.2 右侧面板

进入任一连接命令后，右侧显示 `选项-连接`，至少包括:

- 名称
- 默认名称按连接类型独立编号；若用户尚未自定义名称，切换连接类型时默认名称应同步刷新为该类型下一个可用编号
- 类型
- 图标大小
- 智能推理
- `零件 1`
- `零件 2`
- `零件 2 = Ground`
- 固定副在 `零件 1` 已选后，支持通过三维空白处或模型树 `Ground` 直接完成与大地固定
- 标准模式下的 `位置 / 方向 / Z 轴反向`
- 连接创建态支持 `Alt + 鼠标滚轮` 快速调节图标大小
- `添加 / 取消`

### 6.3 模型树与三维

- 创建成功后在 `连接` 分类下出现 joint 节点
- 连接对象在三维中有独立可选实体
- 连接创建态使用白色预览 shape，创建成功后切换为浅蓝色 shape
- 固定副、转动副、平移副、圆柱副、球副、万向节、螺旋副、平面副都应显示各自独立形状
- 连接节点图标应与对应 Ribbon 连接按钮使用同款图标
- 树选中与三维选中保持同步

## 7. 核心交互流程

### 7.1 闪电模式

1. 用户点击某个连接按钮。
2. 右侧出现 `选项-连接`，默认处于闪电模式。
3. 用户在三维中依次拾取 `零件 1`、`零件 2`。
4. 当鼠标悬停可拾取特征时，三维中显示白色连接预览；完成 `零件 1` 拾取后，连接对象的位置和姿态默认由 `零件 1` 决定。
5. 若开启智能推理，系统根据 `零件 1` 的拾取结果给出位置和方向。
6. `零件 2` 仅用于补全第二参与方，不应把连接位置重新挪到两零件中点。
7. 第二次拾取完成后立即创建连接，并切换为浅蓝色 shape。
8. 创建后仍保持连接创建状态，支持连续创建。

### 7.2 标准模式

1. 用户在 `选项-连接` 中切换到标准模式。
2. 用户拾取 `零件 1`、`零件 2`，或把 `零件 2` 设置为 `Ground`；固定副若通过三维空白处或模型树 `Ground` 选择大地，则直接完成创建。
3. 系统填充草稿位置和方向，默认取自 `零件 1` 的拾取结果。
4. 用户可修改位置、方向，并切换 `Z 轴反向`。
5. 用户点击 `添加` 后正式创建连接。

### 7.3 Ground

1. 用户先拾取 `零件 1`。
2. 用户可通过 `零件 2 = Ground`、模型树 `Ground` 节点，或在固定副创建时单击三维空白处来选择 `Ground`。
3. 系统把第二参与方设为 `Ground`。
4. 固定副若通过三维空白处或模型树 `Ground` 触发，应立即创建并切换到已创建色态；按钮路径仍保持原有双模式逻辑，即闪电模式立即创建、标准模式等待用户确认。

## 8. 功能需求

### 8.1 FR-01 连接类型规范化

- 系统必须把 Ribbon、小部件和导入导出的连接类型统一为小写 canonical value
- 系统必须让连接默认名称按连接类型分别编号，并优先使用该类型下一个可用编号，而不是跨类型共用一条全局编号

### 8.2 FR-02 双模式

- 系统必须支持 `闪电模式 / 标准模式`
- 默认模式为 `闪电模式`

### 8.3 FR-03 参与方拾取

- 系统必须支持依次确定 `零件 1 / 零件 2`
- 系统必须阻止 `零件 1 === 零件 2`
- `Ground` 只能作为 `零件 2`
- 固定副在 `零件 1` 已选后，系统必须支持通过三维空白处或模型树 `Ground` 将第二参与方解析为 `Ground` 并立即完成创建

### 8.4 FR-04 智能推理

- 开启时，系统根据拾取结果推导连接方向
- 关闭时，方向退回世界坐标系 `+Z`

### 8.5 FR-05 姿态编辑

- 标准模式下必须可编辑:
  - 位置
  - 方向
  - `Z 轴反向`

### 8.6 FR-06 模型树与三维可视化

- 系统必须在模型树中显示连接节点
- 系统必须在三维中显示 joint 对象
- 系统必须为 8 类连接渲染可区分的线框形状
- 系统必须在创建态显示白色预览，在创建成功后显示浅蓝色对象
- 系统必须让连接节点图标与对应 Ribbon 连接按钮保持一致
- 树选中和三维选中必须双向同步

### 8.7 FR-07 属性与删除

- 选中连接后必须能查看基本属性
- 系统必须支持保存连接属性修改
- 系统必须支持删除连接

### 8.8 FR-08 数据持久化

导出 connector 时至少保留:

- `name`
- `connectorType`
- `part1`
- `part2`
- `position`
- `direction`
- `iconSize`
- `inferenceEnabled`
- `zAxisReversed`

## 9. 异常与边界

| 场景 | 期望处理 |
|---|---|
| 未选择 `零件 1` 就试图创建 | 阻止并提示 |
| 未选择 `零件 2` 就试图创建 | 阻止并提示 |
| 选择同一零件作为两侧 | 阻止并提示 |
| 把 `Ground` 作为 `零件 1` | 阻止并提示 |
| 智能推理关闭 | 方向回退到世界 `+Z` |
| 第二参与方为 `Ground` | 允许创建 |
| 固定副已选 `零件 1` 后点击三维空白处或模型树 `Ground` | 将第二参与方设为 `Ground` 并立即创建 |

## 10. 本次实现边界

本次代码落地重点是建立 `SC02` 的最小闭环:

- 连接创建状态机
- 双模式切换
- `零件 1 / 零件 2` 拾取
- `Ground` 支持
- 固定副通过三维空白处 / 模型树 `Ground` 直达大地固定
- joint 三维可视化
- 8 类连接线框形状与草稿/已创建颜色态
- 模型树连接节点按 jointType 复用 Ribbon 图标
- 连接默认名称按 jointType 分类编号
- joint 树选/三维选/属性/删除
- connector 导入导出扩展
- 连接创建态的 `Alt + 鼠标滚轮` 图标大小快捷调节

帮助页中的完整特征规则矩阵、万向节双叉精细语义、以及连接选中后零件红蓝高亮，保留为后续增强项。

## 11. 验收标准

### 11.1 P0

- 点击任一连接按钮后出现 `选项-连接`
- 闪电模式下拾取两个参与方后立即创建连接
- 标准模式下必须点击 `添加` 才创建
- `零件 2 = Ground` 可成功创建连接
- 固定副在拾取 `零件 1` 后，可通过三维空白处或模型树 `Ground` 直接创建与大地的固定副
- `零件 1` 选定后，joint 的默认位置和姿态由 `零件 1` 决定
- 连接创建态显示白色 shape，创建成功后切换为浅蓝色 shape
- 创建后模型树出现连接节点，三维出现 joint 对象
- 同类型连接创建时名称按该类型下一个可用编号生成，不同类型从各自编号起算
- 选中连接后可查看属性并删除

### 11.2 P1

- 8 类连接在三维中显示各自独立形状
- 模型树连接节点图标与所选连接类型的 Ribbon 图标一致
- 标准模式修改位置、方向、`Z 轴反向` 后能同步更新三维
- 连接创建态支持 `Alt + 鼠标滚轮` 实时调整图标大小
- 导出后 connector 含姿态与图标字段
- 重新导入后连接仍可显示和选择
