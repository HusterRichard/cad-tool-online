# SC06 PLAN: 添加驱动

## 1. 文档信息

- 场景 ID: `SC06`
- 场景名称: 添加驱动
- 项目: `CadToolOnline`
- 文档版本: `v1.1`
- 最近更新: `2026-05-07`

## 2. 目标

把当前仓库中已有的 motion 骨架能力，收敛为一个可交付的 `SC06 MVP`：

- 从 Ribbon 进入 `选项-驱动`
- 支持 `闪电模式 / 标准模式`
- 能基于连接创建旋转驱动或平移驱动
- 支持属性编辑、模型树、删除、导入导出闭环

## 3. 现状摘要

当前仓库中已有：

- Ribbon `转动驱动 / 平移驱动` 按钮
- `motion` 运行时实体
- 模型树 `驱动` 分类
- `motionProperties` 入口
- `motion` 导入导出基本结构

当前主要缺口：

- 创建态与选择态未完全闭环
- `选项-驱动` 需要按帮助页重构
- 缺少驱动与连接类型兼容性校验
- 属性面板需要对齐帮助页字段
- `Esc / Del / 模型树右键` 需要对驱动完整生效

本次缺陷收口聚焦：

- 驱动创建态拾取转动副时没有三维预览
- `createMotion` 点击链路误走面拾取，导致错误提示 `No face found at click position...`

## 4. 实施范围

### 4.1 本次实现

- 驱动创建态
- `闪电模式 / 标准模式`
- 连接驱动兼容性校验

- 帮助页截图级 UI 复刻

## 5. 技术方案

### 5.1 状态模型

在 `src/webview/main.ts` 中统一维护：

- `motionCreationPanelActive`
- `motionCreationMode`
- `motionDraft`
- `motionCreationPreset`
- `pendingMotionIconSize`

目标是把“驱动草稿”和“已创建驱动实体”分离。

本次补丁额外约束：

- 驱动草稿和已创建驱动改由独立 `MotionVisualizer` 通道渲染
- 已创建驱动默认显示为红色，选中态仍保持红色系强调
- `createMotion` 模式下只允许 joint 选择，不再进入 `resolveFacePlacement`
- 平移驱动使用代码绘制的轴向单箭头轮廓，箭头直接沿 joint 轴生成
- 转动驱动使用代码绘制的弧形箭头轮廓，并让箭头头部由两条边线闭合，同时通过稳定绕轴朝向让顶部保持朝上、箭头头部稳定落在右下象限

### 5.2 选择联动

- 为驱动节点补齐 `selectionKey`
- 统一 `SelectionKind` 支持 `motion`
- 选中驱动时打开 `属性-驱动`
- 创建态下选中连接时，重绘 `选项-驱动`，避免属性面板串台

### 5.3 创建流程

1. 用户点击 Ribbon `转动驱动 / 平移驱动`
2. 打开 `选项-驱动`
3. 默认 `闪电模式`
4. 用户在三维视图 hover/选中连接时，显示与 joint 锚定的驱动预览
5. 执行兼容性校验
6. 闪电模式立即创建；标准模式等待 `添加`
7. 成功后刷新模型树并选中该驱动

### 5.4 兼容性规则

- 旋转驱动:
  - 允许 `revolute`
  - 允许 `cylindrical`
- 平移驱动:
  - 允许 `prismatic`
  - 允许 `cylindrical`

校验失败时不写入实体，只保留当前草稿并提示用户。

### 5.5 属性与删除

- `属性-驱动` 保留 `类型` 只读
- `驱动类型`、`图标大小`、`phi.start`、`w.start` 可编辑
- 删除连接时，级联删除引用该连接的驱动
- `Del` 对纯驱动选中场景直达 `deleteMotion`

### 5.6 导入导出

导入兼容：

- `driveMode`
- `functionType`
- `visible`
- `visibility`
- `phiStart`
- `start`
- `wStart`
- `w`

导出统一输出：

- `driveMode`
- `iconSize`
- `visible`
- `phiStart`
- `wStart`

## 7. 风险与应对

### 7.1 风险

- 创建态下选择连接会被属性面板覆盖
- 连接删除后驱动残留为脏引用
- 旧配置文件中的字段命名不统一
- 驱动类型与连接类型不匹配导致错误配置被导入

### 7.2 应对

- 选择后强制回绘 `选项-驱动`
- 删除连接时级联删除引用驱动
- 导入层做字段别名兼容
- 创建与导入两侧都做兼容性校验
- 在 motion 创建态通过 `selectionFilter` 只允许 joint 命中，避免 motion shape 反向拦截 joint 拾取
- 将驱动绘制逻辑独立到 `MotionVisualizer`，避免继续侵入 `JointVisualizer`

## 8. 完成定义

满足以下条件即认为 `SC06` 实现完成：

1. 创建、属性、删除、导入导出四条主链路可用。
2. `Esc / Del / 右键菜单` 行为完整。
3. 驱动与连接兼容性规则已落地。
4. `node --test test/step-import-selection.test.mjs` 通过。
5. `pnpm build:webview` 通过。
6. 驱动创建态选择 joint 时出现预览，且不再出现错误面拾取提示。
7. 平移驱动预览与已创建图形中的单向箭头和所选平移副保持平行共轴。
8. 转动驱动预览与已创建图形显示为代码绘制的弧形箭头轮廓，箭头头部为双线闭合，且顶部朝上、头部稳定位于右下象限。
9. 已创建驱动在三维视图中显示为红色。
