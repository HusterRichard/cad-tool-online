# SC01 方案与开发计划：添加接触

日期: 2026-03-11
项目: CadToolOnline
主题: 接触设计（Build Contacts）

## 1. 背景

根据帮助文档 `HowToUseCADToolBox/BuildContacts.html`，接触设计不是单一“新增一个 force 节点”的占位功能，而是一整套多体设计工作流，至少包含：

- 接触入口
- `选项-接触` 面板
- 闪电模式
- 标准模式
- 三维视图连续拾取两个零件/特征
- 模型树中的接触节点管理
- 属性查看
- 删除
- 配置导入导出

当前仓库在接触方向仍停留在“Ribbon 图标占位”层，因此 `SC01` 需要先完成 MVP 级闭环。

## 2. 当前实现现状

### 2.1 已有能力

- Ribbon 上已经存在两个接触相关图标入口
- 模型树已有 `力` 分类
- `src/language` 已经为 `contact` 预留 schema / 知识库定义
- Webview 已具备：
  - 三维视图选中零件
  - Ribbon 动作分发
  - 右侧 options / properties 面板渲染框架
  - 配置导入导出主链路

### 2.2 关键缺口

- 接触按钮错误复用了流体命令
- 运行时没有 `MbsContactEntity`
- 没有 `选项-接触` 和 `属性-接触`
- 没有接触创建、删除、连续创建逻辑
- 模型树 `力` 分类当前承载的是流体对象，不是接触对象
- 配置导入导出没有 `contact`
- 帮助页要求的复杂面型校验与红蓝双描边还没有底层能力支撑

## 3. 本次开发目标

### 3.1 业务目标

在在线版中补齐 `SC01` 的最小可用闭环，使用户能够：

- 从 Ribbon 进入接触设计状态
- 在 `选项-接触` 中配置名称、类型、图标大小和模式
- 在三维视图中连续拾取两个零件创建接触
- 在模型树 `力` 分类下管理接触对象
- 查看接触属性、删除接触
- 导出并重新导入接触配置

### 3.2 技术目标

- 将接触从“占位按钮”升级为运行时一等实体
- 让接触对象接入模型树、属性面板、配置导入导出
- 在不大改 three/geo 底层的前提下完成 MVP

## 4. MVP 设计决策

### 4.1 帮助页目标态与 MVP 差异

帮助页目标态要求按球面 / 圆柱侧面 / 矩形面 / 圆面严格限制特征选择，并对两个关联零件做红蓝双描边。当前仓库暂不具备完整面型识别与双对象差异高亮能力。

因此，本次采用如下 MVP 决策：

- 保留 `闪电模式` 与 `标准模式`
- 保留三维拾取两个对象创建接触
- 接触类型先支持当前在线版明确暴露的两类:
  - `point-point`
  - `point-surface`
- 接触记录两次拾取位置，作为接触图标定位依据
- 复杂面型校验写入后续迭代，不阻塞本次交付
- 红蓝双描边写入后续迭代，不阻塞本次交付

### 4.2 数据模型

建议在 Webview 中新增：

```ts
interface MbsContactEntity {
  id: string;
  name: string;
  contactType: 'point-point' | 'point-surface';
  part1: string;
  part2: string;
  pickPoint1: Vec3;
  pickPoint2: Vec3;
  iconSize: number;
  visible: boolean;
  createdAt: string;
}
```

### 4.3 UI 设计

- Ribbon:
  - `点点接触`
  - `点面接触`
- 右侧面板:
  - `选项-接触`
  - `属性-接触`
- 模型树:
  - `力` 分类中新增 contact 子节点
- 三维视图:
  - 通过绿色接触图标/标识表达接触对象

## 5. 实施步骤

### 5.1 Phase 1：入口与实体接线

- 修正 Ribbon 按钮 action id，不再复用流体命令
- 在 `handleMbsAction()` 中增加 `createContact_*`
- 新增 `MbsContactEntity` 存储

完成标准：

- 点击接触按钮能够进入接触设计状态

### 5.2 Phase 2：选项面板与创建流程

- 实现 `选项-接触`
- 增加接触草稿状态
- 实现 `闪电模式` 与 `标准模式`
- 接入三维拾取两个零件创建对象

完成标准：

- 用户可完成一次接触创建
- 用户可在不退出状态的前提下连续创建

### 5.3 Phase 3：模型树、属性与删除

- 接触写入模型树 `力` 分类
- 选中接触显示 `属性-接触`
- 删除接触并同步刷新三维视图与模型树

完成标准：

- 接触具备“创建 -> 查看 -> 删除”闭环

### 5.4 Phase 4：导入导出

- 扩展 `CadtoolConfigExportData`
- 导出 `contact`
- 导入 `contact`
- 未知引用输出 warning

完成标准：

- `contact` 可完成导出后再导入恢复

## 8. 风险与约束

- 若拾取仍基于“零件级 + 点击位置”而不是“面级实体”，则帮助页中的特征类型约束只能先以文档声明差异。
- 若 three viewer 不扩展临时差异高亮能力，则红蓝双描边只能在后续迭代实现。
- 若后续把接触和真实求解语义绑定，当前 `contactType` 枚举及配置字段可能需要再次扩展。

## 9. 交付标准

- 代码中存在真实的接触创建命令，不再走流体占位逻辑
- 用户可通过 Ribbon + 三维拾取创建接触
- 接触可在模型树中看到，可查看属性，可删除
- 接触可导出并导入恢复
- 文档中明确记录帮助页目标态与本次 MVP 差异
