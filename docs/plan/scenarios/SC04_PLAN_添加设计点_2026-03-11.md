# SC04 PLAN: 添加设计点

## 1. 文档信息

- 场景 ID: `SC04`
- 场景名称: 添加设计点
- 项目: `CadToolOnline`
- 文档版本: `v1.1`
- 日期: `2026-05-11`

## 2. 目标

把当前“选中零件后点击面立即创建设计点”的窄流程，升级为与帮助页一致的 `SC04 MVP`：

- 明确 `选项-设计点`
- 支持 `拾取 / 计算`
- 支持显式 `添加`
- 支持模型树 / 属性 / 删除 / 导入导出闭环

## 3. 现状摘要

当前仓库中已有：

- Ribbon `设计点` 按钮
- 面拾取能力
- `mbsDesignPoints` 运行时存储
- `designPoint` 导入导出字段

当前缺口：

- 无草稿态与显式确认
- `计算模式` 未落地
- 无模型树节点闭环
- 无 `属性-设计点`
- `Esc` / `Del` 对设计点不完整

## 4. 实施范围

### 4.1 本次实现

- 设计点创建态改为草稿工作流
- `拾取模式`
- `计算模式`
- `中点 / 偏移 / 受限交点`
- 模型树节点
- 属性面板
- `Del` 删除
- `Esc` / 再次点击 Ribbon / 关闭面板退出
- 导入导出保持兼容

### 4.2 本次不做

- 全量高级几何推理
- 专用设计点渲染器
- 全局统一设计元素尺寸设置入口
- 与连接/标架的自动放置联动

## 5. 技术方案

### 5.1 状态模型

在 `src/webview/main.ts` 中新增设计点工作流状态：

- `designPointDraft`
- `designPointCreationMode`
- `designPointCalcMode`
- `designPointPickIntent`
- `pendingDesignPointSize`

目标是把“创建前草稿”和“创建后实体”分离。

### 5.2 创建流程

1. Ribbon 点击 `设计点`
2. 校验当前已有选中零件
3. 创建空草稿并打开 `选项-设计点`
4. 三维拾取只更新草稿
5. 点击 `添加` 后才写入 `mbsDesignPoints`
6. 写入后刷新模型树，并保留工作流用于连续创建

### 5.3 计算模式

- `中点模式`: 两次点拾取后求平均位置
- `偏移模式`: 基于参考点、方向、偏移值、反向开关
- `交点模式`: 基于可推理的线/面特征，支持 `线-线`、`线-面`

### 5.4 模型树与选择

- 设计点节点挂到所属零件或分组下
- 新增 `designPoint` selection key
- 支持树选中和 viewer 同步

### 5.5 属性与删除

- 新增 `属性-设计点`
- 支持保存名称、位置、方向、图标大小、偏移参数
- `Del` 删除设计点

### 5.6 兼容性

- 保持现有 `designPoint` 导入导出 schema 不破坏
- 删除标架或参考标架时，设计点 `markerRefId` 继续按弱引用处理

## 8. 风险与应对

### 8.1 几何推理能力不足

- 风险: 在线版当前只具备有限的面拾取和特征推理能力
- 应对: 本次把交点模式限制为已有推理结果可覆盖的 `线-线`、`线-面`

### 8.2 视觉样式差异

- 风险: 设计点 3D 外观与桌面版不一致
- 应对: 先复用现有 frame 可视化，后续再独立产品化

### 8.3 依赖声明不稳定

- 风险: `main.ts` 直接使用 `three`，根包未声明会导致构建不稳定
- 应对: 在根包补齐 `three` 依赖，保证 `pnpm build:webview` 稳定通过
